Capture data through Studio & create personalised conversations

In a bot conversation, every bot session is mapped to a record in User 360. When building a bot using Studio, you can use Prompt nodes to capture User properties and store it in the database.

It can store responses of conversations that involve asking for user details such as name, email, phone number, location, or any system/custom user property in the relevant field. You can also enrich user data and create personalised conversations using User properties in Studio conversations.

note

You need to use the userId property to identify a user in a conversation.

Capture userId through conversations​

To update userId of customers through bot conversations:

  1. When building flows in Studio, use the Prompt node or Variables node.

  2. Click Make prompt smarter and scroll-down to the Additional section.

  3. Check Use this as Unique ID for user.

Capture user properties through conversations​

The user record is updated in the database using the session mapping.

  • In the bot Builder, use Prompt or Variable nodes to capture user properties (such as name, email, phone number, location, or any system/custom user property) through conversations.

  • To extract data from a payload, use the Input node and store information in the relevant user property. For example, fetch user details from the user add event payload and store them in relevant user properties.

note

The data captured through Studio conversations for Identified Users will be updated automatically in the Users table.

Fetch user properties through conversations​

You can fetch user properties in conversations using the Text node by clicking on the variable dropdown and selecting any User Property manually. Instead, you can also use {{{user.propertyName}}}.

You can also use user properties to improve engagement rates by creating personalised content.

For example, Hi {{{variables.user_name}}}, you have unlocked an exclusive offer. Shop today and get flat 50% off.

The property value is fetched from the user record mapped to the current session.
